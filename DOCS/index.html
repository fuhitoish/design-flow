デザイン制作フロー

制作に入る前に「何を・どう作るか」を完全に見える状態にする。
手戻りのない、再現性のあるデザインプロセス。

パンフレット・Web・ポスター・バナーなど、あらゆる単一制作物に適用できる「コア ＋ 着せ替えモジュール」型のワークフローです。

5つの設計原則

コア ＋ モジュール

共通の根幹に、制作物別の専用プロセスを着せ替える。無駄なく、漏れなく。

原稿ロック関門

ダミーテキストでのデザイン進行を禁止。実原稿確定後にレイアウトへ。

理由の記録

色・フォント・レイアウトの各決定に理由を残し、ブランド資産として蓄積。

遡及を許容

矛盾が見つかったら該当 Step に戻る。失敗ではなく正常なフロー。

既存ブランド例外

ガイドラインがあるブランドは Step 4 を適用・スキップ選択可。

コア・フロー（Step 1〜4）

制作物の種類に関わらず、制作に入る前に完全ロックする根幹プロセス。各 Step のゲートをすべてクリアしてから次へ進みます。

1

プロジェクト要件（目的とターゲット）

「何のため・誰に・どんな制約で作るか」を明確にする
  • プロジェクト名・制作背景・最終ゴール（1〜2文）
  • ターゲット層の明確化（ペルソナ：年代・職業・課題・価値観）
  • 制作物の種類の選択（後続モジュールを決定）
  • 媒体・仕様の初期確定（未確定は「要確認」と記録）
  • 必須掲載要素のリストアップと優先順位（1〜3）
  • 制約・ルール（使わない色・避けるトーン・参考にしない競合）
  • ビジュアル参考画像 3〜5 枚

→ すべてチェックで Step 2 へ

2

リサーチ・戦略分析

ターゲット・競合・トレンドを理解し、表現の土台を作る
  • ペルソナの深堀り（Step 1 より詳しい理解を 3〜5 点抽出）
  • 競合・参考企業 5〜10 社のビジュアル分析（色・フォント・トーン・レイアウト）
  • 「我々がやってはいけないこと」の明文化
  • トレンド調査（取り入れる／無視するの判断含む）
  • インスピレーション画像 20〜30 枚以上

→ すべてチェックで Step 3 へ

3

コンテンツの完全ロック（原稿と素材） 最重要関門

手戻り最大原因「原稿・素材の未確定」をここで終わらせる
ダミーテキストでのレイアウト設計・デザイン進行は禁止
  • キーメッセージの策定（1〜2文。以降すべての判断基準）
  • ビジュアル方向性の決定（形容詞 2〜3 個 + ムードボード 3〜5 枚）
  • キーワード 3〜5 個の抽出
  • 必須掲載要素ごとの 実原稿（テキストデータ） の確定
  • 支給画像・必須ロゴなど素材の確保
  • テキスト量と各セクションの文字数見積もり

→ 実原稿・素材が揃い、キーメッセージが言語化されている → Step 4 へ

4

ビジュアル・ファウンデーション（基本トンマナ）

色・フォント・画像方向・グリッド・余白を確定。以降は変更しない

4-1 カラーパレット

  • メイン・サブ・アクセント・ニュートラルの決定
  • RGB / CMYK 両値 + 各色の理由づけ
  • Illustrator 等での色見本作成

4-2 タイポグラフィ

  • 見出し・本文フォントの決定
  • サイズ階層（h1 / h2 / h3 / 本文 / キャプション）
  • 和文・洋文・数字混在の表示確認

4-3 画像・ビジュアル方向

  • 画像タイプ（写真 / イラスト / グラフィック）の決定
  • 色・雰囲気・トーンの言語化 + 参考画像 5〜10 枚

4-4 グリッド・余白

  • グリッドシステム（印刷: マージン・段組 / Web: 列数・BP・ガター）
  • ホワイトスペースの戦略（1〜2段落で言語化）

→ 曖昧な項目ゼロ → モジュール（Step 5）へ

着せ替えモジュール（Step 5）

Step 1 で選んだ制作物種別に応じ、1 モジュールのみを適用します。

モジュール A

印刷物・パンフレット

物理制約とページものの情報設計に特化

  • 仕様確定（サイズ・ページ数・用紙・折り・加工）
  • 台割設計（全ページの役割と原稿割り当て）
  • ワイヤーフレーム（紙面・実寸グリッド）
  • 画像スペース指定
  • Step 4 の適用確認
  • 入稿前確認（CMYK・塗り足し・トンボ）
モジュール B

Web サイト・LP

画面可変性とインタラクションに特化

  • 仕様確定（BP・CMS・動作環境・主要機能）
  • 情報設計（サイトマップ・導線）
  • ワイヤーフレーム（各 BP での配置・並び替え）
  • インタラクション設計（ホバー・メニュー・フォーム）
  • CMS 仕様（該当時）
  • Step 4 の適用確認
モジュール C

ポスター・単一広告

一瞬の視認性と設置環境に特化

  • 仕様確定（掲出場所・サイズ・縦横比）
  • 環境シミュレーション（遠距離視認性・コントラスト）
  • 視線誘導設計（Z の法則・メインコピー配置）
  • ワイヤーフレーム（実寸・原稿収まり確認）
モジュール D

バナー・SNS 画像

狭小面積での情報伝達と媒体制約に特化

  • 仕様確定（プラットフォームサイズ・容量上限）
  • セーフゾーン確認（UI 被りエリア）
  • 要素の断捨離（原稿を極限まで削る）
  • ワイヤーフレーム（媒体別レイアウト）

制作・実装（Step 6）

Step 5 で確定した設計に従い、Illustrator / InDesign / HTML・CSS 等で実装します。

やること

  • 確定レイアウト・原稿・素材の配置
  • Step 4 のカラー・タイポグラフィの適用
  • 細部調整（字詰め・色微調整・折り等）
  • 納品ファイルの作成（PDF / 入稿 / 公開）

やらないこと

  • 大幅なレイアウト変更
  • 色・フォントの根本的な変更
  • ページ／セクション構成の変更

上記が必要になった場合は Step 5（または該当コア Step）の詰め不足 と判断。制作を止めて遡及する。

最終ゲート — 「完全に見えている状態」か？

モジュール完了後、制作（Step 6）に進む前に以下をすべて YES にしてください。

  • キーメッセージを自分で述べられる
  • ターゲット層について具体的に話せる
  • 各色・フォントについて理由を述べられる
  • ページ／セクションごとの「役割」を説明できる
  • 仕様変更の質問に、設計理由で答えられる
  • 実装を「ほぼ想像」できる（制作開始時の迷いがない）
✅ すべて YES → 制作ステップへ進行可能
❌ 1 つでも NO → 該当 Step に遡及
