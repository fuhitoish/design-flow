## 1. プロジェクト概要

### 1.1 解決したいこと

制作開始後のイメージズレ・レイアウト変更・色の手戻りを、**計画段階で決め切る**ことで防ぐ。

### 1.2 最終ゴール

計画完了時点で、トンマナ・構成・配置・原稿の置き場所が **制作者・関係者全員に同じイメージで共有** でき、Step 6（制作）では決定に従うだけの状態。

### 1.3 スコープ

| 含む | 含まない（当面） |
|---|---|
| 単一制作物の計画〜制作〜振り返り | 複数案件の PM・工数管理 |
| パンフ・Web・ポスター・バナー（モジュール A〜D） | 動画・3D・大規模 CMS |
| ブラウザ試用アプリ → 本番 Web アプリ | クラウド同期・チーム同時編集 |

## 3. プロセス要件（変更しない骨格）

試用の結論: **Step の順序と関門の考え方は維持**。UX とツール機能を変える。

```
Step 1 要件定義 → Step 2 リサーチ → Step 3 コンセプト・仮原稿
→ Step 4 トンマナ ★ → Step 5 レイアウト（モジュール）★
→ 本番原稿ロック → 最終ゲート ★ → Step 6 制作・納品
→ （将来）振り返りフェーズ
```

### 3.1 ライフサイクル（v1.2 追加要件）

| フェーズ | 内容 |
|---|---|
| **plan** | Step 1〜最終ゲートまで |
| **produce** | Step 6・制作中の進捗・変更リクエスト |
| **retro** | 納品後の振り返り・次案件への学び |

### 3.2 設計原則（Revise）

| # | 原則 |
|---|---|
| R1 | フローは変えない、体験を変える |
| R2 | チェックは結果、入力が本体（自動連動） |
| R3 | ビジュアルファースト（色・配置・原稿は UI で） |
| R4 | 質問に答えるだけ（選択・ピッカー優先） |
| R5 | 制作フェーズまでツールに含める |
| R6 | 用語は常に説明可能（ツールチップ・図解） |
| R7 | モジュール A（印刷物）から実装 |

## 5. デザイン知識（DPL）要件

案件ごとにゼロから調べるのではなく、**ツールが最初から持つ型メニュー**。

| 層 | 内容 | 状態 |
|---|---|---|
| research/ | 色理論・グリッド・視線・印刷のテキスト整理 | ✅ |
| patterns/ | JSON + SVG（P0 15 型） | ✅ v0.1 |
| trial 接続 | Step 4/5 で型を選びテンプレ反映 | ✅ |
| app 本格接続 | カラーピッカー・WF と連動 | ⬜ R2 |

**P0 型の例:** 60-30-10、Z/F 視線、8P 台割、2 ポケットフォルダ、白銀比 √2 など  
詳細: [knowledge/research/07-pattern-candidates.md](../knowledge/research/07-pattern-candidates.md)

## 6. データ要件

### 6.1 案件（Project）— 目標スキーマ

```
Project
├─ lifecyclePhase: plan | produce | retro
├─ fields: Step 1〜6, copy（テキスト同期用）
├─ extras: 構造化 UI データ
│   ├─ colorRoles, typo, personas, copySections
│   ├─ spec, pageMapPages, wireframeZones, imageSpec
│   ├─ copyAreas, tools, deliverables, productionQ
│   └─ changeRequests, refImages, moodUrls, …
├─ appliedPatterns: DPL 適用履歴
├─ checks: 自動連動チェック状態
├─ gateAnswers / gateMemo / gateLog
└─ retroLog
```

### 6.2 永続化

| 環境 | 方式 |
|---|---|
| pilot/trial | ブラウザ `localStorage` + JSON エクスポート |
| app（将来） | DB / ファイル同期（未定） |

## 10. 次回試用の成功基準

- [ ] チェックを **手動で全部 ON にしなくても** Step 完了できる
- [ ] 色を **スウォッチで確認** しながら Step 4 を終えられる
- [ ] 「配置ラフ」「一番伝えたいこと」を **ヘルプなしで** 説明できる
- [ ] 台割上で **原稿の置き場所** が指定できる
- [ ] Step 6 で **制作進捗** を記録できる
- [ ] 型ライブラリから **3 型以上** を適用してテンプレが使える
