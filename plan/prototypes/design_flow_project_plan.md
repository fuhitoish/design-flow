# デザイン標準プロセス確立 & システム構築
## 単一制作物ごとの「完全決定フロー」

**対象者：** Fuhito（マーケティング部門、グラフィック・ウェブ制作）  
**対象制作物：** パンフレット、ウェブサイト、ポスター、バナーなど（単一制作物）  
**目標：** 計画段階で「何をどう作るか」が完全に見える状態を実現し、制作段階での軌道修正をなくす

---

## 1. 本プロジェクトの目的

### 1.1 なぜこれが必要か
```
【現状の課題】
制作開始 → Illustrator で作ってみる 
  → 「あ、イメージと違った」「文字が入らない」「レイアウト変更」
  → 手戻り・作り直し

【目指す状態】
計画段階で色・フォント・構成を完全決定
  → 制作開始（決定に従うだけ）
  → イメージズレなし、軌道修正なし
```

### 1.2 このドキュメントの役割
- **業界標準のデザインプロセス** を言語化・体系化
- Fuhito が **そのプロセスを実装できるように** チェックリスト化
- 最終的には、このフローを **ウェブシステム化**（Phase 2）

## 3. 共通フロー詳細（Step 1 - Step 4）

このセクションは、グラフィック・ウェブ両方で同じプロセスを踏みます。

### Step 1: ブリーフ・要件定義

**目的：** 制作物がなぜ必要か、誰に、どういう制約で作るのかを明確にする

#### 実施内容

##### 1-1. 制作背景と目的
```
【決めること】
- 制作物の名前（パンフレット名、サイト名など）
- 企画背景：なぜ必要か
- 最終ゴール：この制作物を通じて、クライアント・自社は何を達成したいのか
- 期待効果：制作後、どうなることを望んでいるか
  例）認知度向上、コンバージョン増加、ブランドイメージ刷新

【形式】
制作シートに記入、または短い段落で言語化
```

##### 1-2. ターゲット層の定義
```
【決めること】
- メインターゲット（詳細な人物像）
  - 年代、職業、年収、ライフスタイル
  - 関心事、課題、価値観
- サブターゲット（存在すれば）
- ターゲットがこの制作物に期待すること

【やり方】
「ペルソナ」を作成
- 実在する架空の人物をひとり想定
- その人の目線で「このパンフを見たら何を感じるか」を想像
```

##### 1-3. 対象媒体・仕様
```
【グラフィックの場合】
- 制作物の種類（パンフレット、ポスター、チラシなど）
- サイズ（A4、A3、B5など）
- ページ数 / 折り方（二つ折り、三つ折りなど）
- 用紙質（マット、グロス、特殊紙など）
- 印刷色（4色、2色、モノクロ）
- 特殊加工（箔押し、PP加工など）
- 納期・部数

【ウェブの場合】
- 制作内容（ブランドサイト、ランディングページ、リニューアルなど）
- 対応デバイス（PC、タブレット、モバイル）
- CMS（WordPress、カスタム、静的サイトなど）
- 既存サイトの有無（リニューアル vs 新規）
- 予定している機能（ブログ、お問い合わせフォーム、予約機能など）
- ホスティング環境
- 納期
```

##### 1-4. 情報・コンテンツの棚卸し
```
【決めること】
- 必ず含める情報（必須要素）
  例）会社情報、商品画像、アクセス、お問い合わせ先
- 情報の優先度（どれを大きく見せるか）
- テキスト量の見積もり（短い？大量？）
- 画像素材の数・傾向

【形式】
リスト化 or マインドマップ
優先度を1-3で付ける
```

##### 1-5. 制約・ルール
```
【決めること】
- 色の制限（「この色は使わない」など）
- フォントの制限（「ゴシックのみ」など）
- 避けるべき表現・トーン
- 参考にしてはいけない競合企業
- その他の制約

【ビジュアル参考资料】
「こういう感じで」という参考画像を3-5枚集める
```

#### Step 1 チェックリスト
```
☐ 制作物名・背景・目的が明記されている
☐ 最終ゴールが言語化されている（1-2文）
☐ ターゲット層が「具体的な人物像」として記述されている
☐ 媒体・仕様が確定している（曖昧な部分は「要確認」と記録）
☐ 必須情報がリスト化されている
☐ 優先度が付けられている
☐ 制約・ルールが明記されている
☐ ビジュアル参考画像が選定されている

すべてチェック → Step 2 へ進行可能
```

---

### Step 2: リサーチ・戦略分析

**目的：** ターゲット、競合、市場トレンドを理解し、「どう表現するか」の土台を作る

#### 実施内容

##### 2-1. ターゲット理解の深堀り
```
【実施すること】
- Step 1 で定義したペルソナについて、さらに詳しく調べる
- ターゲットが「どういう情報ソースを見ているか」
- ターゲットの「言語」「価値観」「好む表現」
- ターゲットが競合企業・類似サービスをどう見ているか

【方法】
- SNS（Twitter、Instagram）での会話を読む
- 競合サイトのコメント・レビューを見る
- オンラインコミュニティで会話を観察
- アンケート（可能であれば）

【成果】
「このターゲットについて、自分たちが知らなかったこと」を3-5個抽出
```

##### 2-2. 競合・参考企業の分析
```
【実施すること】
- 同じターゲットに訴求している競合企業を5-10個リストアップ
- それらの「ビジュアル表現」「トーン」「メッセージング」を分析
- 「良い事例」と「避けるべき事例」を分類

【具体的には】
- 色使い：暖色系？寒色系？
- フォント：高級感？親しみやすさ？
- 画像：写真？イラスト？
- レイアウト：シンプル？複雑？
- メッセージング：理性的？感情的？

【成果】
競合分析シート（表形式で色・フォント・トーン等を比較）
```

##### 2-3. トレンド・インスピレーション収集
```
【実施すること】
- 現在のデザイントレンド（色、フォント、レイアウト）を調査
- Pinterest、Behance、AWWWARDS など、デザイン参考サイトで検索
- Step 1 で集めた参考画像との関連性を見る

【チェック】
- トレンドを取り入れるべきか、それとも「無視すべき」か
- クライアントのブランドとトレンドのズレはないか
```

#### Step 2 チェックリスト
```
☐ ターゲット層について、Step 1 より詳しい理解が得られた
☐ 競合企業 5-10 社のビジュアル分析が完了
☐ 「我々は何をしてはいけないか」が明確になった
☐ トレンド調査が完了
☐ インスピレーション画像が20-30枚以上集まった
☐ これらを「制作戦略」に落とし込む準備ができた

すべてチェック → Step 3 へ進行可能
```

---

### Step 3: コンセプト開発・ビジュアル戦略

**目的：** 「このターゲットに、この方向性で表現する」というコンセプトを言語化・ビジュアル化

#### 実施内容

##### 3-1. キーメッセージの策定
```
【決めること】
このデザイン全体を通じて、ターゲットに最も伝えたいメッセージ（1-2文）

例）
「京都の歴史と現代性を融合した、上質な体験」
「忙しい30代サラリーマンが、短時間で結果を出せるツール」
「子どもたちが安心して遊べる、親も満足する空間」

【重要性】
全ステップの判断基準になる
「これはキーメッセージに合致しているか」で色・フォント・レイアウトを決める
```

##### 3-2. ビジュアル方向性の決定
```
【決めること】
- 全体的な「雰囲気」「トーン」
  - 上品 / カジュアル / 力強い / 親しみやすい などの形容詞
  - 複数組み合わせてもOK（例：「上品だが、親しみやすい」）

- 視覚的な「世界観」
  - 自然？都市？？古典？現代？
  - 色のトーン：明るい？暗い？落ち着いている？

【やり方】
ムードボード的に、参考画像を3-5枚に絞り込む
それらを見て、共通する「空気感」を言語化
```

##### 3-3. キーワード抽出
```
【決めること】
キーメッセージとビジュアル方向性から、3-5個のキーワードを抽出

例）「京都 Miyabi 案件」の場合：
- 「歴史」「上品」「洗練」「静寂」「京都」

→ これが、Step 4 のビジュアルエレメント選択の指針になる
```

#### Step 3 チェックリスト
```
☐ キーメッセージが1-2文で言語化されている
☐ ビジュアル方向性が形容詞で説明されている
☐ ムードボード（参考画像）が3-5枚選定されている
☐ キーワードが3-5個抽出されている
☐ これらすべてが、Step 1 のターゲット・目的と整合している

すべてチェック → Step 4 へ進行可能
```

---

### Step 4: ビジュアルエレメント確定

**目的：** 「色」「フォント」「画像方向」「グリッド」「白間」など、実装に必要な全てを決定する

**このステップが、「計画段階で完全に見える状態」の最後の砦。ここで決まったものが、制作段階では変わらない。**

#### 実施内容

##### 4-1. カラーパレット決定

**やり方：**
```
Step 1: トーン分析
- Step 3 のムードボード（参考画像）から、色をすべて抽出
- どういう色が「この世界観」に使われているか観察

Step 2: 色選定
- Adobe Color / 配色ツール を使って、候補を作成
- 3-5色に絞り込む
  - メインカラー（1色）
  - サブカラー（2-3色）
  - アクセント（1色）
  - ニュートラル（黒・グレー・白）

Step 3: 理由づけ（重要）
各色について、「なぜこの色を選んだか」を言語化
例）
  「メインカラーは深緑（RGB 60, 100, 80）
    理由：京都の自然、奥深さ、上品さを表現しつつ、
    現代的な落ち着きを感じさせる色」

Step 4: 実装値の確定
RGB（ウェブ） / CMYK（印刷）の両値を記載

Step 5: Illustrator で色見本を作成
実際に並べて、調和・組み合わせを確認
  （Fuhito が既にやっている方法）
```

**成果物：**
```
カラーパレット仕様書
├─ メインカラー：#～、RGB (  ,  ,  )、CMYK (  ,  ,  ,  )、理由
├─ サブカラー①：～
├─ サブカラー②：～
├─ アクセント：～
├─ ニュートラル（黒、グレー、白）の指定値
└─ 色見本（Illustrator での並べた画像）
```

##### 4-2. タイポグラフィシステム（フォント）決定

**やり方：**
```
Step 1: 視覚的要件の整理
- 大見出し / 中見出し / 小見出し / 本文 / キャプション
  各レイヤーで何個のフォントが必要か

Step 2: フォント候補選定
- Google Fonts から候補を探す
- 【重要】各レベルで「複数案」を用意
  例）
  大見出し候補：Noto Serif JP（格調高い） / Roboto（モダン）
  本文候補：Noto Sans JP（読みやすさ重視） / Source Sans Pro（洋文向け）

Step 3: 組み合わせ検証
- 候補フォントを Illustrator に打ち込んで、実装イメージで比較
- 和文 / 洋文の混在の見え方を確認
  （特に数字・英字を含む場合、重要）

Step 4: 最終決定
- メインフォント（見出し）
- 副フォント（本文）
- の2-3種類に絞り込む

Step 5: サイズ体系の決定
フォントサイズの「階層」を決める
例）
  h1（タイトル）：36pt
  h2（章見出し）：24pt
  h3（項目名）：16pt
  本文：12pt / 14pt
  キャプション：10pt

この「体系」が、制作物全体の可読性を左右する
```

**成果物：**
```
タイポグラフィシステム仕様書
├─ 大見出しフォント：○○ ◎pt
├─ 中見出しフォント：○○ ◎pt
├─ 本文フォント：○○ ◎pt
├─ ウェイト指定（Regular / Bold / Light）
├─ 行間・字間のルール（例：1.5倍行間）
└─ フォント組み合わせのビジュアル見本
```

##### 4-3. 画像・ビジュアルエレメント方向の決定

```
【決めること】
- 画像の種類：写真？イラスト？グラフィック？混在？
- 写真の場合、どういう「タイプ」か
  - 人物写真？風景？商品？
  - 色合い（彩度高い？低い？くすみ？）
  - 雰囲気（明るい？暗い？）
  
- その他のビジュアル要素
  - グラフィック・パターン・モチーフ
  - アイコン（必要なら）
  - イラストのタッチ（もし使うなら）

【方法】
Step 3 のムードボードから「画像の特性」を分析
Pinterest や Unsplash で、同様の雰囲気の画像を探す
Illustrator で、色・ノイズ・トーンを試す

【成果物】
参考画像 5-10枚 + 「我々が使う画像はこの方向」という指針
```

##### 4-4. グリッド・レイアウト基本方針

```
【決めること】

【グラフィックの場合】
- ページサイズ内での「マージン」（余白）
- 基本グリッド（例：8mm グリッド、または黄金比ベース）
- 段組（1段？2段？3段？）
- 「ページの余白」と「要素間の余白」のリズム

【ウェブの場合】
- グリッドシステム（12列？8列？）
- ブレークポイント（PC: 1200px, Tablet: 768px, SP: 375px など）
- ガター（列間の余白）
- レスポンシブ時の挙動

【方法】
Step 3 のムードボードで「ホワイトスペースの使い方」を観察
「余白が広い」「コンパクト」など、特性を言語化
その特性に合わせて、グリッドを決定

【成果物】
グリッド図示 + マージン・ガター値の記載
```

##### 4-5. ホワイトスペース・リズム戦略

```
【決めること】
- 「余白」をどう使うか
  - 余白を大きく取る → 高級感、読みやすさ
  - 余白を小さく → エネルギッシュ、情報量重視
  
- 要素間の「リズム」
  - 余白が「統一」 → 整然、フォーマル
  - 余白が「変化」 → 動的、アンティーク

【方法】
ムードボードから観察 + 自分たちのターゲット・コンセプトに合わせて決定

【成果物】
「余白の哲学」を1-2段落で言語化 + ビジュアル見本
```

#### Step 4 チェックリスト
```
【カラー】
☐ メインカラー、サブカラー、アクセント、ニュートラルが決定
☐ RGB / CMYK 両値が記載
☐ 各色の「理由づけ」が1-2文で記載
☐ Illustrator での色見本が作成・確認

【タイポグラフィ】
☐ 見出し用フォントが決定（Google Fonts 指定）
☐ 本文用フォントが決定
☐ フォントサイズの階層（h1 = ◎pt など）が定義
☐ 和文・洋文・数字の混在表示が確認

【画像・ビジュアル】
☐ 画像のタイプ（写真／イラスト etc）が決定
☐ 画像の特性（色・雰囲気）が言語化
☐ 参考画像が5-10枚選定

【グリッド・余白】
☐ グリッドシステムが決定（グラフィック向け・ウェブ向け）
☐ マージン / ガター値が記載
☐ ホワイトスペースの戦略が言語化

【全体確認】
☐ 上記すべてが、Step 3 のキーメッセージ・キーワードと整合
☐ Step 1 のターゲット・目的に合致
☐ 制約・ルール（Step 1）に違反していない

すべてチェック → 分岐（Step 5a / 5b）へ進行可能

⚠️ このステップの「詰め」が、制作段階での手戻りを決定する
   1つでも曖昧な項目があれば、進むべきではない
```

## 4. 分岐フロー（グラフィック向け）

### Step 5a: グラフィック情報構成・ページレイアウト設計

**目的：** ページ数、各ページの「何をどこに置く」を完全に決定。Step 4 で決めた色・フォント・グリッドを「適用」する段階。

#### 実施内容

##### 5a-1. ページ構成の設計
```
【決めること】
- 総ページ数
- 各ページの役割・内容
  例）A4 8ページパンフレット
    表1（表紙）：タイトル + ブランド写真
    表2（裏表紙）：企業情報 + QR コード
    1p：導入文 + キービジュアル
    2-3p：見開き、商品説明（テキスト + 画像）
    4-5p：見開き、事例紹介（複数カラム）
    6-7p：見開き、FAQ
    中1-2p：折り込みページ（特典情報など）

【方法】
Step 1 で決めた「必須情報」を、優先度順に配置していく
「このページは何を伝えるのか」を各ページで明確に
```

##### 5a-2. 各ページのレイアウト設定
```
【決めること】
各ページについて、以下を決定：
- 1ページ内の「段組」（1段？2段？3段？）
- 見出しの位置（上部？中央？）
- 画像スペースのサイズ・トリミング方向
- テキストボックスの配置・高さ
- カラムの幅

【重要なチェック】
- テキスト量は、確保したテキストボックスに収まるか
  （ここでの確認が、制作段階での「文字が入らない」問題を防ぐ）

【方法】
Illustrator でラフレイアウトを実制作で作成
テキストボックスの高さを測り、実際のテキスト量と照合
画像スペースを実寸で確認

【成果物】

```

```

【成果物】

```

##### 5a-5. 色・フォント・グリッドの「適用確認」
```

## 6. 「計画段階で完全に見えている状態」の定義

**このセクションが、本プロジェクトの最重要ポイント。**

### 6.1 具体的に「何が見えているのか」

#### グラフィック（パンフレット）の例
```
【完全に見えている状態】

□ ページ構成
  「A4 8ページ、表紙・本文6ページ・裏表紙の構成」
  「表紙は大きな見出しとブランド写真」
  「2-3ページは見開きで、左テキスト・右画像」
  が、頭の中で「画像」として浮かぶ

□ 色
  「メインカラーはこの深緑、サブはこのベージュ、
   アクセントはこのゴールド」
   が、確定している

□ フォント
  「見出しはこのセリフ体、本文はこのゴシック」
  が、確定している

□ レイアウト
  「ページの上下左右のマージンはこのくらい、
   コラムは3段で、余白はこの比率」
  が、設計されている

□ テキスト
  「このセクションにはこのテキスト量が入る」
  ことが確認されている
  （文字が入らない問題は起きない）

□ 画像
  「このスペースには、こういう雰囲気の、
   このサイズの画像が入る」
  が、確定している

【この状態では】
制作は「実装」に徹する
色・フォント・レイアウトに従うだけ
「あ、違う」という軌道修正は発生しない
```

#### ウェブサイトの例
```
【完全に見えている状態】

□ サイト構造
  「ホーム → About → Products（3製品） → ブログ → Contact」
  の階層が見えている

□ 各ページのレイアウト
  「ホームはファーストビュー大きく、その下にセクション3つ」
  「各セクションは画像左、テキスト右」
  が、想像できる

□ レスポンシブ
  「PC では2段、SP では1段に自動で切り替わる」
  が、設計されている

□ 色・フォント
  「全ページで同じ色・フォント体系を使う」
  が、確定している

□ インタラクション
  「ハンバーガーメニューはここに」
  「ボタンはホバーで色が変わる」
  が、仕様化されている

【この状態では】
開発は「仕様に従う」だけ
「ワイヤーフレームにはなかったコンポーネント」を
急に追加することはない
```

### 6.2 「見えている状態」に達するために必要な工数

```
Step 1（要件定義）：1-2日
Step 2（リサーチ）：1-2日
Step 3（コンセプト）：1-2日
Step 4（ビジュアル確定）：2-3日 ← ここに時間をかける
─────────────────────────
計画フェーズ合計：6-9日

その後：
Step 5a（グラフィック構成）：1-2日 or
Step 5b（ウェブ構成）：2-3日
```

### 6.3 「完全に見えている状態」の品質チェック

以下のすべてに「YES」と答えられれば、OK：

```
□ キーメッセージを自分で述べられるか
  （「この制作物は、結局何を伝えてるの？」と聞かれて、
   1-2文で答えられるか）

□ ターゲット層について、具体的に話せるか
  （「これ、誰に見せるの？」と聞かれて、
   その人の年代・職業・課題を説明できるか）

□ 色についての理由を述べられるか
  （「なぜこの色？」と聞かれて、
   「このターゲットに、この雰囲気を感じさせるため」と答えられるか）

□ ページ（またはセクション）ごとの「役割」を説明できるか
  （各ページ / セクションで何を伝えるのか、
   細かく説明できるか）

□ 「もし仕様変更が出たら？」という質問に答えられるか
  （例：「もし見出しサイズを大きくしたら、このセクションのレイアウトは？」
   → 「このセクションはテキストボックスに余裕があるので対応可能」
   のような「設計理由」を説明できるか）

□ 実装を「ほぼ想像」できるか
  （制作を始めた時、「あ、このレイアウトどうしよう」という
   迷いが発生しないか）
```

## 9. 最後に：このフローの効果

```
【仮説】

今までのワークフロー：
要件定義（1日）
  ↓
曖昧なコンセプトで即座に制作開始
  ↓
Illustrator で作る → 「あ、違った」「文字入らない」「色微妙」
  ↓
手戻り・修正（3-5日）
───────────────
合計：4-6日


このフロー（提案）：
要件定義（1日）
  ↓
リサーチ・コンセプト（2-3日）
  ↓
ビジュアルエレメント完全確定（2-3日）
  ↓
情報構成・ワイヤーフレーム（1-2日）
  ↓
制作実装（2-3日） ← ここは「決定に従うだけ」
───────────────
合計：8-12日

【見た目は長いが】
- 計画段階の時間は「再利用可能」（修正が少ない）
- 制作段階での手戻りが「ほぼなし」
- 最終成果物の「品質」が安定
- 「なぜこの色なのか」という理由が全て言語化されている
  → 後年、同じブランドで追加制作する時の資産になる

Bellagio / Bellagio Miyabi など、ブランドごとの
「デザインガイドライン」として蓄積される
