# Design Flow 用語集

アプリ UI・ラベル・ヘルプテキストの基準。Revise プラン [REVISE_PLAN.md](./REVISE_PLAN.md) §4.3 より。

| 用語（内部） | UI ラベル | 説明 |
|---|---|---|
| Brief | 要件・背景 | なぜ今作るか・誰のためか |
| Key Message | 一番伝えたいこと | この制作物の核心メッセージ（1〜2 文）。以降の判断基準 |
| Copy Draft | 仮原稿 | 文案は未完成で OK。字数・段落・見出しの有無は本番想定 |
| Copy Master | 本番原稿 | 制作直前の確定文案 |
| WF / Wireframe | 配置ラフ（ワイヤーフレーム） | 色や写真の前に、箱と文字だけで配置を決める図 |
| Page Map | 台割 | 各ページの役割と、どの原稿をどこに載せるか |
| Image Spec | 写真の置き場 | 画像ボックスのサイズ・比率・トリミング方向 |
| Tonmana | トンマナ | 色・フォント・ビジュアルの統一ルール |
| Gate | 関門 | 次の Step に進んでよいかの確認ポイント |
| Oral Memo | 確認メモ | 最終ゲートで自分に説明できたことのメモ（任意） |
| Deliverable | 納品物 | クライアントや印刷会社に渡す最終ファイル |
| Retro | 振り返り | 納品後のレビューと次案件への学び |

## 廃止・変更するラベル（pilot → app）

| 旧（pilot） | 新（app） |
|---|---|
| キーメッセージ（説明なし） | 一番伝えたいこと |
| 口述メモ | 確認メモ |
| WF | 配置ラフ |
| 納品物（説明なし） | 納品物（例: 入稿 PDF） |
