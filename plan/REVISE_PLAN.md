## 3. 設計方針（Revise の原則）

試用フィードバックを踏まえ、v1.2 以降の実装は以下を原則とする。

| # | 原則 | 説明 |
|---|---|---|
| R1 | **フローは変えない、体験を変える** | Step 1〜6 + 原稿 + ゲートの骨格は PLAN v1.1 を維持 |
| R2 | **チェックは結果、入力が本体** | チェックボックスは「入力完了の表示」に降格 |
| R3 | **ビジュアルファースト** | 色・レイアウト・原稿はキャンバス/プレビューで操作 |
| R4 | **質問に答えるだけ** | 長文テキストより、選択・スライダー・ピッカー優先 |
| R5 | **制作フェーズをツールに含める** | 計画終了 ≠ ツール終了。Step 6 + 振り返りまで |
| R6 | **用語は常に説明可能** | WF 等は初出時にツールチップ + 図解 |
| R7 | **モジュール A から実装** | 今回試用はパンフ系。B〜D は A 完成後 |

## 4. 横断機能の実装計画

### 4.1 チェックリストと入力の連動

**対応 FB:** X-01, X-05, stepNotes.5

**現状:** チェックと入力が独立。全部手動 ON。

**実装:**

```
各チェック項目
  └─ 対応するデータフィールド（必須/任意）を定義
  └─ フィールドが充足 → 自動チェック ON
  └─ 未充足 → チェック不可（グレーアウト）
  └─ 「この Step を完了」は全必須チェック自動 ON 時のみ有効
```

**成果物:**

| 層 | 内容 |
|---|---|
| plan | チェック ↔ フィールド対応表を PLAN.md 付録に追加 |
| pilot | trial に最低限の連動（必須 3 項目だけ）を試験導入 |
| app | 全 Step でルールエンジン実装 |

**優先度:** P0（最優先）

---

### 4.2 ガイド型入力ウィザード

**対応 FB:** X-04, P-03, S4-06

**実装:**

- 各 Step を「1 画面 1 質問」または「1 画面 1 カード」に分割
- 回答タイプ: 単一選択 / 複数選択 / 短文 / 数値 / ファイル
- 例（Step 6）: 「制作時のこだわりは？」→ 字詰め / 色の微調整 / 折り・加工 のチェック式

**成果物:** app の Step UI コンポーネント `GuidedField`

**優先度:** P1

---

### 4.3 用語ヘルプとラベル改善

**対応 FB:** X-06, S3-01, S5-03, S5-04, C-04, P-02, G-01〜03

**用語集（アプリ内表示）:**

| 用語 | 表示ラベル案 | 説明（1 文） |
|---|---|---|
| WF | ワイヤーフレーム（配置ラフ） | 色や写真を入れる前の、箱と文字だけの配置図 |
| キーメッセージ | 一番伝えたいこと | この制作物で観客に残す核心メッセージ（1〜2 文） |
| 口述メモ | 声に出して確認したメモ | 最終ゲートで自分に説明できたかの記録（任意） |
| 納品物 | クライアントに渡すもの | PDF・入稿データ・印刷会社への渡し物など |
| 画像スペース指定 | 写真を入れる場所のサイズ | 各ページで画像ボックスの大きさと比率 |
| 本番原稿ロック | 文案の最終確定 | 仮原稿を本番テキストに差し替えて制作 GO |

**成果物:** `plan/glossary.md` + アプリ内 `?` ツールチップ

**優先度:** P0（文言のみ先行可能）

## 5. ライフサイクル拡張

### 5.1 ライフサイクル拡張

**対応 FB:** X-02

**現状のフェーズ:**

```
計画（Step 1〜5 + 原稿 + ゲート）→ Step 6 → 終了
```

**Revise 後:**

```
┌─────────────────────────────────────────────────────────┐
│ Phase A: 計画（Plan）                                    │
│   Step 1〜5 + 仮原稿 + トンマナ + WF 確定                │
│   → 本番原稿ロック → 最終ゲート                          │
├─────────────────────────────────────────────────────────┤
│ Phase B: 制作（Produce）                                 │
│   Step 6 — ツール選択・進捗・版管理・変更リクエスト対応   │
├─────────────────────────────────────────────────────────┤
│ Phase C: 振り返り（Retro）                               │
│   納品後レビュー・手戻り分析・次案件への資産化            │
└─────────────────────────────────────────────────────────┘
```

**PLAN.md への反映:** §8 Step 6 を「制作」、新 §8.1「振り返り」を追加（v1.2 で正本更新）

**優先度:** P1（計画書更新は P0、ツール実装は P1）

## 8. データモデル拡張（app 向け）

PLAN §10.1 を以下に拡張する。

```
Project
├─ phase: plan | produce | retro
├─ core: Step 1〜4（構造化フィールド）
├─ module: Step 5（A: pageMap[], wireframes[], specs{}）
├─ copy: sections[] → areas[]（割付連動）
├─ gate: checks[] + memos{}
├─ production: tools[], progress{}, versions[]
├─ retro: lessons[], assetsToReuse[]
├─ documents[]（ファイル参照）
├─ assets[]（画像・ムードボード）
└─ personas[]（ペルソナカード）
```

## 9. 次回試用の成功基準

次に同じ案件（または別案件）で試用するとき、以下を満たせば Revise 成功とする。

- [ ] チェックを**手動で全部 ON にしなくても** Step 完了できる
- [ ] 色を**スウォッチで確認**しながら Step 4 を終えられる
- [ ] 「WF」「キーメッセージ」を**ヘルプを見ずに**説明できる（ラベル改善）
- [ ] 台割上で**原稿の置き場所**が指定できる
- [ ] Step 6 で**制作進捗**を記録できる
- [ ] 遡及が必要なとき**変更リクエスト**から Step に戻れる
