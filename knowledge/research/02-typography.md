# 02 — タイポグラフィ・文字組み

Step 4 でフォント階層・和欧混植を**型として選び、プレビューしながら確定**するための整理。

---

## 1. タイポグラフィの役割

| 役割 | 内容 |
|---|---|
| 階層 | 何を最初に読ませるか（見出し > 小見出し > 本文） |
| トーン | 堅い / 親しみ / 高級 / 伝統 |
| 可読性 | 特に高齢者・小さい本文（試用案件で重要） |
| ブランド一貫 | ロゴ・既存ガイドラインとの整合 |

---

## 2. 文字サイズ階層の型

### 2.1 モジュラー・スケール（比率でサイズを決める）

ベースサイズ（例: 本文 10pt）に対し、比率で見出しを決める。

| 比率名 | 倍率 | 用途例 |
|---|---|---|
| マイナー・セカンド | 1.067 | 控えめな階層 |
| メジャー・セカンド | 1.125 | Web でもよく使う |
| パーフェクトフォース | 1.414（√2） | **和・印刷で試用時に言及** |
| 黄金比 | 1.618 | 見出しと本文のメリハリ |
| オクターブ | 2.0 | 表紙タイトルなど大胆な差 |

**DPL 候補:** ✅ P0 — 「スケールプリセット」として 3〜4 種

### 2.2 固定階層（印刷パンフでよくある）

| 要素 | 目安（A4 本文） | メモ |
|---|---|---|
| 表紙 h1 | 28〜42pt | 試用案件で 48→42pt 遡及あり |
| h2 | 14〜18pt | |
| h3 | 11〜12pt | |
| 本文 | 9〜11pt | 9pt 未満は高齢者に厳しい |
| キャプション | 7〜8pt | |

**DPL 候補:** ✅ P0 — モジュール A 用「A4 パンフ標準階層」

---

## 3. フォントの組み合わせ（ペアリング）の型

### 3.1 セリフ + サンセリフ

- **例:** 見出し＝明朝 / 本文＝ゴシック
- **向く:** 和文の定番・可読性と格式のバランス
- **DPL 候補:** P0

### 3.2 同一ファミリーでウェイト分け

- **例:** Noto Sans JP Regular / Medium / Bold
- **向く:** 統一感・Web フォント運用が楽
- **DPL 候補:** P0

### 3.3 明朝のみ（和・高級）

- **向く:** 旅館・美術・伝統ブランド
- **避ける:** 長文本文・小サイズ
- **DPL 候補:** P1

### 3.4 欧文 + 和文の分離指定

| 要素 | よくある指定 |
|---|---|
| 見出し（和） | 明朝・太め |
| 本文（和） | ゴシック |
| 英数字 | 別フォント（Helvetica, DIN 等）で揃える |
| 数字 | タブular figures（表・価格） |

**DPL 候補:** ✅ P0 — 試用 FB「和欧数字の使い分け」

---

## 4. 日本語組版の実務ルール

| 項目 | ガイドライン |
|---|---|
| 1 行の字数 | 日本語 35〜40 字/行 前後（媒体による） |
| 行間 | 本文 140〜180%（フォントサイズ比） |
| 段組 | 1 段（長文）/ 2 段（情報量多） |
| 禁則処理 | 行頭句点・行末開き括弧を避ける |
| ウェイト | 本文は Regular。細すぎる Light は印刷で弱い |

---

## 5. 可読性チェック（高齢者・富裕層向け）

試用案件（50〜70 歳）向けの型として:

- 本文 **10pt 以上**（9pt は下限）
- コントラスト **本文と地は十分な差**
- 行長 **短め**（1 段 35 字前後）
- 余白 **多め**（詰め込みすぎない）

**DPL 候補:** 「シニア向け可読性プリセット」P1
